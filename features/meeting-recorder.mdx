## Capabilities

- Start and stop meeting recording sessions
- Stream transcript chunks into the UI
- Persist recordings and transcript metadata
- Generate AI analysis and execute suggested actions

## Typical flow

1. Start a recording from the meeting recorder view.
2. Stream audio chunks into the backend.
3. Watch transcript chunks arrive in real time.
4. Stop recording and let Xpdite finish processing.
5. Generate analysis or execute follow-up actions from the saved recording.

- Recordings and transcript metadata are persisted in backend storage.
- Analysis settings can be stored and reused between runs.
