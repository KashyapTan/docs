  - Bun

  - UV

  - Mintlify CLI if you want to preview the docs site
</Info>

## Install dependencies

<Steps>
  <Step title="Clone and install">

  ```bash
  git clone https://github.com/KashyapTan/Xpdite.git
  cd Xpdite
  bun install
  bun run install:python
  ```
  </Step>
  <Step title="Run the app in development">

  ```bash
  bun run dev
  ```

  This starts the Vite renderer, the Electron main process, the Python backend, and the Ollama startup helper. The Channel Bridge starts after the backend passes health checks.
  </Step>
</Steps>

## Run services individually

```bash
bun run dev:react
bun run dev:electron
bun run dev:pyserver
```

## Build and validate

```bash
bun run build
bun run lint
bun run dist:win
```

## Preview the docs site

If you are editing this Mintlify documentation site, run the following commands from the docs repository root:

```bash
npm i -g mint
mint dev
mint broken-links
```

## Engineering conventions

- Keep the API layer thin. Business logic belongs in services.
- Add new UI pages under `src/ui/pages/`, register routes in `src/ui/main.tsx`, and wire navigation from layout components.
- Add REST endpoints in `source/api/http.py` or related API modules and expose matching frontend calls in `src/ui/services/api.ts`.
- Add WebSocket message types only after updating both the server implementation and the documented protocol.
- Add new MCP servers in `mcp_servers/servers/<name>/` and register them through the backend MCP manager.
- Marketplace installs, hook compatibility, and direct package installs are handled by `source/services/marketplace/service.py` and the marketplace endpoints in `source/api/http.py`.

## Testing expectations

- Add regression tests for bug fixes.
- Keep backend tests aligned with `source/` module boundaries.
- Keep frontend tests aligned with `src/ui/` behavior and routing.
- Update documentation in the same PR when a user-facing flow or contract changes.
