## Start using Xpdite

Use this guide if you want to install the desktop app and reach a working first prompt quickly.

<Info>
  **Platform support**
  - Windows 10 or Windows 11
  - Ollama is optional, but required for local models
  - Cloud providers are optional, but require your own API keys
</Info>

<Steps>
  <Step title="Install Xpdite">
    Download the latest Windows build from the [Xpdite releases page](https://github.com/KashyapTan/Xpdite/releases).

    If SmartScreen appears because the app is not code-signed yet, verify that the installer came from the official GitHub releases page before you continue.
  </Step>
  <Step title="Choose your model path">
    If you want local inference, install Ollama and pull the default model:

    ```bash
    ollama pull qwen3-vl:8b-instruct
    ```

    If you want cloud models, open **Settings** after first launch and add a key for Anthropic, OpenAI, Gemini, or OpenRouter.
  </Step>
  <Step title="Send your first prompt">
    Launch Xpdite, wait for the boot screen to finish, then:

    1. Press `Alt + .` to capture a screenshot into the active tab.
    2. Type your request.
    3. Press `Enter` to submit.

    You should see a streaming response, screenshot chips in the active tab, and working settings pages.
  </Step>
</Steps>

## Optional setup

- Connect Google in **Settings** > **Connections** if you want Gmail and Calendar MCP tools.
- Open **Settings** > **Marketplace** if you want to install extra skills, plugins, or MCP bundles.
- Configure terminal approval level if you plan to let the model run shell commands.
- Pair Telegram, Discord, or WhatsApp in **Settings** > **Mobile** if you want remote access.
- Enable only the models you want visible in the app to keep the selector focused.
