## Configuration layers

1. Environment variables
2. Persisted SQLite settings
3. Filesystem-backed data under `user_data/`

## Environment variables

| Variable | Default | Purpose |
|---|---|---|
| `XPDITE_USER_DATA_DIR` | Project `user_data/` in dev | Root for DB, memory, skills, screenshots, and artifacts |
| `XPDITE_SERVER_HOST` | `127.0.0.1` | Backend bind host |
| `XPDITE_SERVER_TOKEN` | Generated by Electron | Token for guarded loopback APIs |
| `XPDITE_THREAD_POOL_SIZE` | `4` | Thread pool size for blocking work |
| `PYTHON_SERVER_PORT` | `8000` hint | Backend port hint before auto-selection |
| `BRIDGE_PORT` | `9000` | Channel Bridge port |
| `XPDITE_MOBILE_DEBUG_LOGS` | unset | Verbose mobile bridge logs when set to `1` |

## User data layout

`XPDITE_USER_DATA_DIR` typically contains:

- `xpdite_app.db`
- `screenshots/`
- `artifacts/`
- `memory/`
- `skills/builtin/`
- `skills/user/`
- `skills/preferences.json`
- `mobile_channels_config.json`
- `google/token.json` after OAuth

## Security-sensitive config

- Artifact routes require loopback access and may require `X-Xpdite-Server-Token`.
- API keys are encrypted before storage.
- The renderer runs with Electron security defaults such as `contextIsolation`.
